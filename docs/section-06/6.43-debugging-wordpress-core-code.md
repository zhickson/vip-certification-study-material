# 6.43 Debugging WordPress Core Code - Study Notes

## Overview

Stepping through and inspecting core files involves tracing execution paths in core, adding temporary debug hooks to core (safely), and contributing patches for reproducible bugs.

**Key Reference:** [Xdebug - Section 6.4](./6.4-xdebug.md) | [Debugging with Actions/Filters - Section 6.12](./6.12-debugging-with-actions-filters.md)

---

## 1. Tracing Execution Paths in Core

### Xdebug

**Step Through Core:**
- Set breakpoints in core
- Step through execution
- Inspect variables
- Understand flow

### Query Monitor

**Core Queries:**
- View core queries
- Check execution
- Identify issues

---

## 2. Adding Temporary Debug Hooks to Core (Safely)

### Don't Modify Core

**Never Edit Core:**
- Updates will overwrite
- Use hooks instead
- Add filters/actions

### Use Hooks

**Add Debug Hooks:**
```php
add_action( 'wp_loaded', function() {
    error_log( 'wp_loaded fired' );
} );
```

---

## 3. Contributing Patches for Reproducible Bugs

### Report Bugs

**WordPress Trac:**
- Create ticket
- Provide steps
- Include code
- Test patches

### Contribute

**Process:**
1. Reproduce bug
2. Create patch
3. Submit to Trac
4. Test fixes
