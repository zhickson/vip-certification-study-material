# 6.7 Browser Developer Tools - Study Notes

## Overview

Browser developer tools are essential for debugging front-end performance and JavaScript issues in WordPress. They provide inspection, debugging, and profiling capabilities directly in the browser.

**Key Reference:** [Chrome DevTools Documentation](https://developer.chrome.com/docs/devtools/) | [Firefox Developer Tools](https://developer.mozilla.org/en-US/docs/Tools)

---

## 1. Inspecting DOM Changes and Styling

### Elements/Inspector Panel

**DOM Inspection:**
- View HTML structure
- Inspect element properties
- Edit HTML in real-time
- See computed styles
- View box model

**Access:**
- Right-click â†’ Inspect
- F12 or Cmd+Option+I (Mac) / Ctrl+Shift+I (Windows)
- Elements tab (Chrome) / Inspector tab (Firefox)

### Style Inspection

**Computed Styles:**
- See all applied CSS
- Identify style sources
- View cascade order
- Check specificity
- See overridden styles

**Box Model:**
- View dimensions
- See padding/margin
- Check positioning
- Visual layout representation

### Live Editing

**Edit Styles:**
- Modify CSS in real-time
- See changes immediately
- Test different values
- Copy working styles

**Edit HTML:**
- Modify DOM structure
- Test layout changes
- See immediate results
- Revert changes easily

## 2. Analysing Network Requests and Load Times

### Network Panel

**Request Overview:**
- List of all network requests
- Request method (GET, POST, etc.)
- Status codes
- Response time
- File size

**Request Details:**
- Headers (request and response)
- Preview of response
- Response body
- Timing breakdown
- Initiator (what triggered request)

### Timing Analysis

**Load Time Breakdown:**
```
Total: 1.2s
  - DNS Lookup: 50ms
  - Initial Connection: 200ms
  - SSL Negotiation: 100ms
  - Time to First Byte: 300ms
  - Content Download: 550ms
```

### Performance Insights

**Identify Issues:**
- Slow requests
- Large file sizes
- Blocking resources
- Unused resources
- Cache issues

### WordPress-Specific Requests

**Common Requests:**
- WordPress core files
- Theme assets
- Plugin assets
- AJAX requests
- REST API calls
- Admin-ajax.php

## 3. Debugging JavaScript Errors in Console

### Console Panel

**Error Messages:**
- JavaScript errors
- Warnings
- Console.log output
- Network errors
- Security errors

**Error Information:**
- Error message
- File and line number
- Stack trace
- Error type
- Context

### Console Commands

**Common Commands:**
```javascript
// Log messages
console.log('Debug message');
console.error('Error message');
console.warn('Warning message');

// Group messages
console.group('Group name');
console.log('Message 1');
console.log('Message 2');
console.groupEnd();

// Table output
console.table(arrayData);

// Time operations
console.time('Operation');
// Code to measure
console.timeEnd('Operation');
```

### Breakpoints

**Set Breakpoints:**
- Click line number in Sources panel
- Set conditional breakpoints
- Set logpoints (log without breaking)
- Pause on exceptions

**Debugging:**
- Step through code
- Inspect variables
- Evaluate expressions
- Modify values
- Continue execution

### Source Maps

**Debug Minified Code:**
- Source maps map minified to original
- See original file names
- See original line numbers
- Set breakpoints in original code

## 4. Performance Profiling

### Performance Panel

**Record Performance:**
- Record page load
- Record user interactions
- Analyse frame rate
- Identify bottlenecks
- Optimize rendering

### Performance Metrics

**Key Metrics:**
- First Contentful Paint (FCP)
- Largest Contentful Paint (LCP)
- Time to Interactive (TTI)
- Cumulative Layout Shift (CLS)
- First Input Delay (FID)

### Flame Chart

**Visual Timeline:**
- See function execution
- Identify slow functions
- See call stack
- Identify performance issues

### Memory Profiling

**Memory Usage:**
- Take heap snapshots
- Compare snapshots
- Identify memory leaks
- Track object allocation
