# 6.26 IDE Customization for Debugging - Study Notes

## Overview

Configuring IDEs for WordPress debugging involves integrating Xdebug, setting breakpoints and debug sessions, and installing WordPress-specific extensions for efficient development.

**Key Reference:** [VS Code PHP Debugging](https://code.visualstudio.com/docs/languages/php) | [PHPStorm Xdebug](https://www.jetbrains.com/help/phpstorm/configuring-xdebug.html)

---

## 1. Integrating Xdebug in IDEs

### VS Code

**Install Extensions:**
- PHP Debug
- PHP Intelephense

**Configuration:**
```json
{
    "version": "0.2.0",
    "configurations": [
        {
            "name": "Listen for Xdebug",
            "type": "php",
            "request": "launch",
            "port": 9003,
            "pathMappings": {
                "/var/www/html": "${workspaceFolder}"
            }
        }
    ]
}
```

### PHPStorm

**Settings:**
- Languages & Frameworks → PHP → Debug
- Configure Xdebug port (9003)
- Set path mappings
- Enable breakpoints

---

## 2. Setting Breakpoints and Debug Sessions

### Breakpoints

**Set Breakpoints:**
- Click line number
- Red dot indicates breakpoint
- Conditional breakpoints
- Logpoints

### Debug Sessions

**Start Debugging:**
- Click "Start Debugging"
- Set breakpoints
- Step through code
- Inspect variables
