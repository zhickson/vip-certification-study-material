# 6.38 Bypassing or Disabling Code/Plugins - Study Notes

## Overview

Isolating issues by disabling components involves temporarily bypassing plugin code, disabling hooks or themes selectively, and confirming sources of conflicts.

**Key Reference:** [Debugging with Actions/Filters - Section 6.12](./6.12-debugging-with-actions-filters.md)

---

## 1. Temporarily Bypassing Plugin Code

### Disable Plugin

**WP-CLI:**
```bash
wp plugin deactivate plugin-name
```

**Rename Plugin:**
```bash
mv wp-content/plugins/plugin-name wp-content/plugins/plugin-name-disabled
```

**Code Bypass:**
```php
// Temporarily bypass
if ( ! defined( 'BYPASS_PLUGIN' ) || ! BYPASS_PLUGIN ) {
    // Plugin code
}
```

---

## 2. Disabling Hooks or Themes Selectively

### Remove Hooks

**Remove Action:**
```php
remove_action( 'init', 'problematic_function', 10 );
```

**Remove Filter:**
```php
remove_filter( 'the_content', 'problematic_filter', 10 );
```

### Switch Theme

**WP-CLI:**
```bash
wp theme activate twentytwentyfour
```

---

## 3. Confirming Sources of Conflicts

### Process of Elimination

**Steps:**
1. Disable all plugins
2. Enable one by one
3. Identify conflict
4. Test with different themes

### Isolate Issues

**Test in Isolation:**
- Single plugin active
- Default theme
- Minimal configuration
- Add components back

---

## Exam Tips

### Key Points to Remember

1. **Bypassing:**
   - Deactivate plugins
   - Rename directories
   - Code bypasses

2. **Disabling:**
   - Remove hooks
   - Switch themes
   - Isolate components

3. **Confirmation:**
   - Process of elimination
   - Test in isolation
   - Identify conflicts
