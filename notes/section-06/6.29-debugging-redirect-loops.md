# 6.29 Debugging Redirect Loops - Study Notes

## Overview

Debugging redirect loops involves tracing redirect logic with headers, inspecting wp_redirect() usage, and identifying plugin or server-level loops.

**Key Reference:** [wp_redirect() - WordPress Codex](https://codex.wordpress.org/Function_Reference/wp_redirect)

---

## 1. Tracing Redirect Logic with Headers

### Check Headers

**cURL:**
```bash
curl -I -L https://example.com
```

**Browser DevTools:**
- Network panel
- Check redirect chain
- View status codes

### Identify Loops

**Symptoms:**
- Multiple redirects
- Circular redirects
- "Too many redirects" error

---

## 2. Inspecting wp_redirect() Usage

### Find Redirects

**Search Code:**
```bash
grep -r "wp_redirect" wp-content/
```

**Common Causes:**
- Incorrect site URL
- SSL/HTTPS issues
- Plugin conflicts
- .htaccess rules

---

## 3. Identifying Plugin or Server-Level Loops

### Plugin Issues

**Check Plugins:**
- Disable plugins one by one
- Check plugin redirects
- Look for conflicts

### Server Issues

**Check .htaccess:**
```apache
# Look for redirect rules
RewriteRule ^(.*)$ https://example.com/$1 [R=301,L]
```

**Check wp-config.php:**
```php
define( 'WP_HOME', 'https://example.com' );
define( 'WP_SITEURL', 'https://example.com' );
```

---

## Exam Tips

### Key Points to Remember

1. **Tracing:**
   - Check headers
   - Follow redirect chain
   - Identify loops

2. **WordPress:**
   - Check wp_redirect()
   - Verify URLs
   - Check config

3. **Server:**
   - Check .htaccess
   - Verify server config
   - Check plugins
