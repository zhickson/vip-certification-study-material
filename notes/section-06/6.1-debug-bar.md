# 6.1 Debug Bar - Study Notes

## Overview

Debug Bar is a WordPress plugin that adds a debug menu to the admin bar, providing quick access to debugging information including database queries, hooks, and request/response details.

**Key Reference:** [Debug Bar Plugin](https://wordpress.org/plugins/debug-bar/)

---

## 1. Viewing Database Queries and Execution Time

### Installation

```bash
wp plugin install debug-bar --activate
```

### Accessing Debug Information

**Admin Bar Menu:**
- Debug Bar appears in the admin toolbar when activated
- Click to expand and view debugging panels
- Available on both frontend and admin pages

### Database Queries Panel

**Features:**
- Lists all database queries executed on the page
- Shows query execution time
- Displays query count
- Shows total query time

**Example Output:**
```
Queries: 45
Total Time: 0.2345s
Slowest Query: 0.0123s
```

### Query Details

**Information Displayed:**
- SQL query text
- Execution time per query
- Query caller (function/file)
- Stack trace for query origin

---

## 2. Inspecting Hooks Firing on Page Load

### Hooks Panel

**Features:**
- Lists all actions and filters fired during page load
- Shows hook execution order
- Displays hook priorities
- Shows callback functions registered to hooks

### Hook Information

**Displayed Data:**
- Hook name
- Type (action or filter)
- Priority
- Callback function
- Execution count

**Example:**
```
init (action)
  - Priority 10: my_plugin_init()
  - Priority 20: another_callback()

the_content (filter)
  - Priority 10: wpautop()
  - Priority 11: shortcode_unautop()
```

---

## 3. Reviewing Request/Response Details in Admin

### Request Panel

**Information Displayed:**
- HTTP method (GET, POST, etc.)
- Request URI
- Query string parameters
- POST data
- Cookies
- Headers

### Response Panel

**Information Displayed:**
- Response headers
- HTTP status code
- Response time
- Memory usage

### PHP Information

**Displayed Data:**
- PHP version
- Memory limit
- Max execution time
- Loaded extensions
- PHP configuration

---

## 4. Using Debug Bar Effectively

### Best Practices

**1. Development Only:**
```php
// Only enable in development
if ( defined( 'WP_DEBUG' ) && WP_DEBUG ) {
    // Debug Bar should be active
}
```

**2. Identify Slow Queries:**
- Look for queries with high execution time
- Check for duplicate queries
- Identify N+1 query problems

**3. Hook Debugging:**
- Verify hooks are firing
- Check hook priorities
- Identify hook conflicts

**4. Request Analysis:**
- Verify request parameters
- Check authentication headers
- Debug AJAX requests

---

## 5. Limitations and Alternatives

### Debug Bar Limitations

- Basic interface compared to Query Monitor
- Limited query analysis features
- No HTTP request inspection
- No conditional debugging

### When to Use Query Monitor Instead

- More detailed query analysis
- Better performance profiling
- HTTP request debugging
- Conditional and template debugging

---

## Exam Tips

### Key Points to Remember

1. **Debug Bar Features:**
   - Database queries and execution time
   - Hooks fired on page load
   - Request/response details
   - PHP information

2. **Access:**
   - Admin bar menu
   - Available on frontend and admin
   - Requires plugin activation

3. **Use Cases:**
   - Quick debugging overview
   - Query performance check
   - Hook verification
   - Request inspection

4. **Limitations:**
   - Basic compared to Query Monitor
   - Limited analysis features
   - No advanced profiling

5. **Best Practices:**
   - Use in development only
   - Combine with Query Monitor for detailed analysis
   - Remove in production
