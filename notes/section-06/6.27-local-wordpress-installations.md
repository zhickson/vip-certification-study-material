# 6.27 Local WordPress Installations - Study Notes

## Overview

Local WordPress installations using wp-env, Local, or Docker allow you to debug WordPress, handle configuration mismatches, and test plugins/themes in isolation.

**Key Reference:** [wp-env](https://github.com/WordPress/gutenberg/tree/trunk/packages/env) | [Local by Flywheel](https://localwp.com/)

---

## 1. Debugging with wp-env, Local, or Docker

### wp-env

**Setup:**
```bash
wp-env start
```

**Debug:**
- Access at localhost:8888
- Enable WP_DEBUG
- Use Query Monitor
- Xdebug support

### Local

**Features:**
- One-click setup
- Built-in tools
- Easy switching
- SSL support

### Docker

**Compose:**
```yaml
services:
  wordpress:
    image: wordpress:latest
    ports:
      - "8080:80"
```

---

## 2. Handling Configuration Mismatches

### Common Issues

**Problems:**
- PHP version differences
- Extension mismatches
- Database versions
- Memory limits

### Solutions

**Match Production:**
- Use same PHP version
- Install same extensions
- Match database version
- Configure limits

---

## 3. Testing Plugins/Themes in Isolation

### Isolation

**Benefits:**
- Test without affecting production
- Isolate plugin conflicts
- Test theme changes
- Debug safely

### Testing

**Process:**
- Install plugin/theme
- Test functionality
- Debug issues
- Fix problems
