# 6.42 PHP Stack Traces - Study Notes

## Overview

Reading stack traces to identify root causes involves interpreting call order in trace logs, mapping traces to source code, and identifying misbehaving functions.

**Key Reference:** [PHP Error Logs - Section 6.3](./6.3-php-error-logs.md)

---

## 1. Interpreting Call Order in Trace Logs

### Stack Trace Format

**Example:**
```
Stack trace:
#0 /path/to/file.php(42): function_name()
#1 /path/to/file2.php(15): another_function()
#2 /path/to/file3.php(8): main_function()
```

**Reading:**
- Bottom to top = call order
- Top to bottom = execution order
- Line numbers show location

---

## 2. Mapping Traces to Source Code

### Find Files

**Locate Files:**
- Check file paths
- Verify line numbers
- Open in editor
- Review code

### Understand Flow

**Trace Execution:**
1. Start at bottom
2. Follow call chain
3. Identify issue
4. Fix problem

---

## 3. Identifying Misbehaving Functions

### Common Issues

**Problems:**
- Undefined functions
- Missing parameters
- Type mismatches
- Logic errors

### Debug Process

**Steps:**
1. Read stack trace
2. Identify function
3. Check parameters
4. Review logic
5. Fix issue

---

## Exam Tips

### Key Points to Remember

1. **Call Order:**
   - Bottom to top
   - Line numbers
   - File paths

2. **Mapping:**
   - Find files
   - Check lines
   - Review code

3. **Identification:**
   - Find function
   - Check parameters
   - Review logic
