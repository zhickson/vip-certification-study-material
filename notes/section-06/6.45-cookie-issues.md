# 6.45 Cookie Issues - Study Notes

## Overview

Debugging authentication/session cookie problems involves inspecting cookies in browser tools, debugging cookie expiration and persistence, and resolving conflicts across plugins or domains.

**Key Reference:** [Browser Developer Tools - Section 6.7](./6.7-browser-developer-tools.md) | [Request/Response Headers - Section 6.8](./6.8-request-response-headers.md)

---

## 1. Inspecting Cookies in Browser Tools

### Browser DevTools

**Application Tab:**
- View all cookies
- Check values
- See expiration
- Test deletion

**Network Tab:**
- View cookie headers
- Check Set-Cookie
- Monitor cookie changes

---

## 2. Debugging Cookie Expiration and Persistence

### Check Expiration

**Cookie Properties:**
- Expires date
- Max-Age
- Session vs persistent
- Domain/path

### WordPress Cookies

**Common Cookies:**
- wordpress_logged_in_*
- wordpress_*
- wp-settings-*
- comment_author_*

---

## 3. Resolving Conflicts Across Plugins or Domains

### Domain Conflicts

**Issues:**
- Cross-domain cookies
- Subdomain problems
- Path conflicts

### Plugin Conflicts

**Check Plugins:**
- Cookie-setting plugins
- Authentication plugins
- Cache plugins

### Fix Conflicts

**Solutions:**
- Adjust cookie domain
- Change cookie path
- Disable conflicting plugins
- Use different names

---

## Exam Tips

### Key Points to Remember

1. **Inspection:**
   - Browser DevTools
   - Application tab
   - Network tab

2. **Expiration:**
   - Check dates
   - Session vs persistent
   - Domain/path

3. **Conflicts:**
   - Domain issues
   - Plugin conflicts
   - Name collisions
