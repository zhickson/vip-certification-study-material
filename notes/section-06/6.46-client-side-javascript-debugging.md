# 6.46 Client-Side JavaScript Debugging - Study Notes

## Overview

Debugging JS issues in WordPress themes/plugins involves using browser console and breakpoints, debugging script dependencies and load order, and tracing front-end rendering bugs.

**Key Reference:** [Browser Developer Tools - Section 6.7](./6.7-browser-developer-tools.md)

---

## 1. Using Browser Console and Breakpoints

### Console

**View Errors:**
- JavaScript errors
- Warnings
- Console.log output
- Network errors

### Breakpoints

**Set Breakpoints:**
- Click line number
- Conditional breakpoints
- Step through code
- Inspect variables

---

## 2. Debugging Script Dependencies and Load Order

### Check Dependencies

**Network Panel:**
- View script loading
- Check dependencies
- Verify order
- Identify conflicts

### WordPress Enqueue

**Check Enqueuing:**
```php
// Check dependencies
wp_enqueue_script( 'my-script', 'url', array( 'jquery' ), '1.0', true );
```

---

## 3. Tracing Front-End Rendering Bugs

### Rendering Issues

**Common Problems:**
- DOM not ready
- Missing elements
- Timing issues
- Event handlers

### Debug Process

**Steps:**
1. Check console
2. Set breakpoints
3. Inspect DOM
4. Trace execution
5. Fix issue

---

## Exam Tips

### Key Points to Remember

1. **Console:**
   - View errors
   - Check logs
   - Test code

2. **Breakpoints:**
   - Set in code
   - Step through
   - Inspect variables

3. **Dependencies:**
   - Check loading
   - Verify order
   - Fix conflicts
